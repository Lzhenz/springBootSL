# 1.项目用途

    当前项目用于springboot的练手，为了熟悉后端的整体构造以及加深自身的印象。作为自己的第一个springboot项目。

# 2.技术栈

    前端：vue

    后端：springboot + mybatis

    构建工具：maven

# 3.问题记录

## 3.1 第一次check

**Controller 层问题：**

1. **❌ 硬编码依赖** - 在 Controller 中 [new SearchPhoneService()](vscode-file://vscode-app/c:/HSCN/Microsoft%20VS%20Code/resources/app/out/vs/code/electron-browser/workbench/workbench.html)，应该使用 Spring 的依赖注入
2. **❌ 业务逻辑耦合** - Controller 直接调用 Service 取值，然后在 Controller 中提取字段，应该让 Service 返回完整的对象或结果
3. **❌ 错误处理缺失** - 没有异常处理，没有返回明确的响应格式
4. **❌ 参数硬编码** - `"xiaomi"` 这个值硬编码在方法中，应该作为请求参数
5. **❌ 返回值类型不统一** - 直接返回 String，最好返回统一的响应格式（如 `ResponseEntity` 或 `Result` 对象）

**DTO 层问题：**

1. **⚠️ 命名不规范** - 应该改成 `PhoneDto`（驼峰写法，不用全大写 DTO）
2. **⚠️ 没有默认构造函数显示** - Lombok 生成了，但建议明确
3. **⚠️ 缺少 toString/equals/hashCode** - 可以让 Lombok 生成


## **Service 层问题：**

1. **❌ 没有 @Service 注解** - 不能被 Spring 管理
2. **❌ 不能被依赖注入** - 因为没有注解
3. **❌ 业务逻辑过于简单** - 实际项目中应该有数据库查询、业务规则等
4. **❌ 没有接口** - Service 最好面向接口编程
